# Helo 登录 / 注册页面 UI 设计文档

## 1. 这份文档解决什么问题

首页现在已经按 Manychat 的营销页气质完成了：强视觉、强节奏、强转化。

但登录 / 注册页不是营销页，它的任务不是继续说服用户，而是让用户快速、安全、清楚地进入产品。所以认证页不继续像素级复刻 Manychat，而是基于首页已经形成的 Helo 风格，做一套更适合产品入口的界面。

这份文档后续会作为 `/login` 和 `/register` 的实现依据，避免实现时“凭感觉设计”。

## 2. 总体设计结论

登录 / 注册页采用：

- Helo 自己的简洁产品入口风格
- 延续首页的品牌色、字体气质、按钮质感和柔和背景
- 不复刻首页的大型营销区块
- 不做复杂动效
- 不堆砌插画和说明文字
- 优先保证 Better Auth 登录注册链路清晰可用

一句话：

> 首页负责吸引用户，认证页负责让用户顺利进入产品。

## 3. 和首页的关系

认证页需要继承首页的这些视觉特征：

- 品牌名：Helo
- 核心黑色文字：`#121212`
- 柔和米白背景：`#f5f1ea`
- 亮黄绿色强调色：`#d8ff58`
- 紫色 / 品红只做少量点缀，不大面积使用
- 按钮使用圆角胶囊感，但尺寸比首页 CTA 更克制
- 字体保持厚实、清晰、有一点海报感，但表单文字必须易读

认证页不继承首页的这些东西：

- 不使用超大 Hero 标题
- 不使用满屏营销文案
- 不使用聊天框大动效
- 不使用多屏滚动叙事
- 不使用复杂背景人物图作为主视觉

## 4. 页面范围

本阶段只设计三个页面 / 状态：

- `/login`：登录页
- `/register`：注册页
- 登录后跳转目标：`/dashboard`

暂时不做：

- 忘记密码
- 邮箱验证码页
- Google 登录
- 多因素认证
- 复杂 onboarding
- 付费墙

这些可以等基础认证链路跑通后再加。

## 5. 整体布局

### 桌面端布局

页面采用居中单列表单结构。

推荐尺寸：

- 页面最小高度：`100svh`
- 主容器宽度：`min(100% - 40px, 1120px)`
- 表单卡片宽度：`420px`
- 表单区域垂直居中，但整体略微上移，避免显得下坠
- 顶部左侧保留 Helo Logo，可点击回首页

结构：

```text
页面背景
├── 顶部 Logo / 返回首页
└── 居中认证面板
    ├── Helo 小标识
    ├── 标题
    ├── 一句说明
    ├── 表单
    ├── 主按钮
    └── 登录/注册链接
```

### 移动端布局

移动端不做左右分栏，直接单列。

推荐尺寸：

- 页面左右留白：`20px`
- 表单卡片宽度：`100%`
- 表单卡片最大宽度：`420px`
- 顶部 Logo 与表单之间保持 `56px` 到 `72px`
- 输入框高度保持 `52px`

## 6. 背景设计

背景要和首页有联系，但不能抢表单注意力。

建议：

- 基础背景：`#f5f1ea`
- 左上角或右下角加入非常轻的色块过渡
- 使用低透明度色彩：
  - lime：`rgba(216, 255, 88, 0.28)`
  - purple：`rgba(138, 62, 232, 0.10)`
  - coral：`rgba(244, 127, 67, 0.12)`
- 背景模糊半径：`48px` 到 `72px`

不要：

- 不做玻璃拟态大面板
- 不做深色全屏背景
- 不直接复用首页导师大图
- 不用复杂图案干扰输入框

## 7. 认证面板

面板是页面核心。

推荐规格：

- 宽度：`420px`
- 背景：`rgba(255, 255, 255, 0.78)`
- 边框：`1px solid rgba(18, 18, 18, 0.10)`
- 圆角：`18px`
- 内边距：桌面 `34px 34px 30px`
- 阴影：`0 24px 70px rgba(18, 18, 18, 0.10)`
- 背景模糊：`blur(16px)`

注意：

这个面板可以有轻微“半透明纸张感”，但不要做成很强的玻璃卡片。首页之前已经反复调整过质感，认证页要更稳。

## 8. Logo 与品牌区

Logo 推荐放两个位置：

1. 页面左上角：作为回首页入口
2. 表单内部顶部：作为品牌识别锚点

左上角 Logo：

- 文字：`Helo`
- 字号：`24px`
- 字重：`900`
- 颜色：`#121212`
- 点击跳转：`/`

表单内部小标识：

- 可以使用一个小圆形 / 方形品牌点
- 尺寸：`38px × 38px`
- 背景：`#d8ff58`
- 文字：`H`
- 圆角：`12px`
- 字重：`900`

## 11. 按钮设计

按钮要和首页 CTA 有关系，但尺寸收敛。

主按钮：

- 高度：`54px`
- 宽度：`100%`
- 圆角：`999px`
- 背景：`#121212`
- 文字颜色：`#ffffff`
- 字号：`14px`
- 字重：`900`
- 字母间距：`0.02em`
- 顶部间距：`22px`

hover：

- 背景：`#000000`
- transform：`translateY(-1px)`
- 阴影：`0 12px 28px rgba(18, 18, 18, 0.18)`

loading：

- 按钮文字改为：
  - 登录中...
  - 创建中...
- 禁用重复点击
- 透明度：`0.72`

禁用状态：

- 背景：`rgba(18, 18, 18, 0.28)`
- cursor：`not-allowed`

## 13. 错误与提示状态

错误提示：

- 出现在对应字段下方，或按钮上方的统一错误条
- 背景：`rgba(244, 127, 67, 0.12)`
- 边框：`rgba(244, 127, 67, 0.28)`
- 文字颜色：`#8a2f13`
- 圆角：`10px`
- 内边距：`10px 12px`
- 字号：`13px`
- 字重：`750`

常见错误：

- 邮箱格式不正确
- 密码不能为空
- 密码至少 8 位
- 邮箱或密码错误
- 该邮箱已被注册

成功提示：

- 注册成功后直接进入 `/dashboard`
- 不单独做成功页

## 14. 动效

认证页动效必须轻。

页面进入：

- 面板从 `opacity: 0; transform: translateY(10px)` 到正常状态
- 时间：`260ms`
- easing：`ease-out`

按钮 hover：

- `180ms`

错误提示出现：

- 淡入即可
- 不抖动

不做：

- 不做聊天框滚动
- 不做大面积视差
- 不做复杂插画动画

## 17. 实现边界

第一版实现只做：

- `/login`
- `/register`
- 表单 UI
- 基础前端校验
- Better Auth email/password 调用
- 成功后跳转 `/dashboard`
- 已登录访问 `/login` 或 `/register` 时跳转 `/dashboard`

第一版不做：

- 第三方登录
- 找回密码
- 邮箱验证流程
- 复杂 onboarding
- 手机号登录
- 社交登录分割线

## 18. 文件规划建议

后续实现时建议这样拆：

```text
app/login/page.tsx
app/register/page.tsx
components/auth/AuthShell.tsx
components/auth/AuthCard.tsx
components/auth/AuthInput.tsx
components/auth/AuthSubmitButton.tsx
app/auth.css
```

解释：

- `app/login/page.tsx`：登录页面，只负责页面入口和登录逻辑。
- `app/register/page.tsx`：注册页面，只负责页面入口和注册逻辑。
- `AuthShell.tsx`：认证页公共背景、Logo、居中布局。
- `AuthCard.tsx`：中间白色表单面板。
- `AuthInput.tsx`：统一输入框样式和错误状态。
- `AuthSubmitButton.tsx`：统一按钮 loading / disabled 状态。
- `app/auth.css`：登录注册页面专用样式，避免污染首页 CSS。

## 19. 验收标准

实现完成后，用这几个标准判断是否合格：

- 打开 `/login`，第一眼能看出这是 Helo，不像外部模板。
- 页面比首页克制，但颜色、字体、按钮质感能接上首页。
- 输入框、按钮尺寸不夸张，能像正式 SaaS 产品。
- 注册和登录路径清楚，没有多余干扰。
- 移动端不挤、不溢出。
- 错误提示能看懂。
- Better Auth 链路能跑通。

## 20. 最终方向

认证页不是第二个首页。

它应该像一个安静、可靠的入口：

用户被首页说服后，来到这里，只需要很清楚地完成一件事：

> 进入 Helo，继续推进自己的目标。
